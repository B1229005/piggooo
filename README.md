#  PIGGO：整合行程管理與社群分享的智慧旅遊應用系統
#  專案概述
  現代人在規劃與執行旅遊時，常面臨「跨多個 App 切換（如地圖、筆記、社群）」所導致的資訊碎片化與效率低落。  
  PIGGO 是一款涵蓋「旅行前、中、後」全方位的智慧旅遊 APP。  
  本系統整合了底層 Provider 狀態管理與 Firebase 即時資料庫，並導入 AI 分析技術，打破傳統旅遊資訊格式不一的壁壘。  
  從個人化旅行人格測驗、群組共編決策，到旅途導航與社群分享，提供完整的一站式完美體驗，徹底解決旅遊資訊難以整合的痛點。

#  核心功能
開始使用：提供流暢的登入註冊流程，並內建「旅行人格測驗」。AI 將根據測驗結果萃取個人偏好標籤，精準推薦適合使用者的專屬行程與景點。

計畫：視覺化的行程安排與共編行李清單。內建決策聊天室，AI 會負責比對店家營業時間與推薦景點順序，使用者可將 AI 推薦選項直接拋轉至群組進行投票，達成民主決策。

首頁：旅途中的數位儀表板。整合當地天氣、群組分帳、交通資訊、下一站導航、翻譯與匯率轉換，以及群組定位分享。AI 會在背景監控天氣變化，主動推播注意事項與旅行節奏提醒。

相機：提供景點相關的參考照片，並具備 AI 拍照指導與構圖建議，協助記錄完美旅程。

我的：專屬的旅遊社群足跡。使用者可發布旅遊貼文、分享個人私房行程，經營個人化旅遊品牌。

交流：雙向社群論壇。包含分享優質行程的「推薦版」，以及解決在地疑難雜症的「交流版」。AI 會自動統整並摘要網友的長篇留言，並進行事實查證 (Fact-checking)。

#  系統使用指南
初始化與測驗：首次開啟 APP 完成註冊後，系統會自動引導進入「旅行人格測驗」，請依直覺作答以利 AI 建立初始偏好模型。

建立群組與計畫：進入【計畫】分頁，邀請好友加入群組。在聊天室中，系統會依據群組成員的偏好交集，推送 AI 行程卡片。

無縫決策：點擊 AI 卡片下方的「加入投票選項」，即可在聊天室內發起表決。確認後的景點將自動同步至團隊行程表。

行中儀表板：旅遊當日切換至【首頁】，系統會依據行程表自動提供下一站導航、當下匯率與分帳工具，並透過 AI 推播天氣警告。

#  實作細節與演算法
無延遲狀態同步：捨棄傳統的 setState，全面導入 Provider 架構。在「行李清單」與「聊天室投票」功能中，任何單一節點的狀態改變，皆能在毫秒級內同步至所有群組成員的畫面上。

AI 降維與結構化處理：系統透過呼叫後端 AI API，將外部社群論壇的非結構化圖文資訊，萃取並轉換為 PIGGO 資料庫可辨識的標準 JSON 格式，大幅降低資料清洗的成本。

動態 UI 渲染：考量到旅遊資訊變動極快，專案大量使用 StreamBuilder 監聽 Firestore 的即時變化，確保營業時間與投票結果的絕對正確性。

功能收斂設計：系統架構設計上嚴格控管冗餘功能，確保操作流程專注於 AI 推薦與群組決策的無縫拋轉，維持極簡的使用者體驗。
