## 我的

| 欄位 | 描述內容 |
| :--- | :--- |
| **使用案例名稱** | **發布旅遊回憶貼文** |
| **行動者** | 使用者、Firebase雲端資料庫 |
| **前置條件** | 使用者已透過第三方登入成功，並處於「我的」頁面。 |
| **正常情節** | 1. 使用者在個人主頁點擊「新增貼文」按鈕。<br>2. 系統開啟編輯介面，使用者選擇手機相簿中的旅遊照片，並輸入文字心得、地點與hashtag。<br>3. 使用者確認內容無誤後，點擊「發佈」按鈕。<br>4. 系統觸發狀態管理（Provider），將圖文資料打包為標準Post資料模型。<br>5. 系統將照片檔案上傳至雲端儲存空間，並將貼文的JSON結構寫入Firestore資料庫。<br>6. 個人主頁的動態牆透過StreamBuilder監聽到資料庫更新，即時渲染並呈現全新貼文。 |
| **例外情節** | **3a. 網路中斷或上傳失敗：**<br>系統捕捉到Firebase連線錯誤，自動將此筆貼文儲存至本地端（Local Storage）作為草稿，並跳出提示：「上傳失敗，已為您保留草稿，請檢查網路連線後重試。」 |
| **後置條件** | 成功將旅遊貼文寫入雲端並即時顯示於「我的」頁面中。 |
