## 地圖與群組定位

| 欄位 | 描述內容 |
| :--- | :--- |
| **使用案例名稱** | **設定與廣播地圖集合點** |
| **行動者** | 使用者、Socket.io (後端) |
| **前置條件** | 系統已成功建立 WebSocket 連線。 |
| **正常情節** | 1. 使用者點擊首頁的「集合！」按鈕，彈出地圖對話框。<br>2. 系統判斷當前無集合點，或使用者為發起人（具備修改權限）。<br>3. 使用者在地圖上點選任意位置，地圖上出現暫時的 Marker。<br>4. 使用者點擊「📍 確認位置」按鈕。<br>5. 系統透過 Socket 發送 `setMeetingPoint` 事件，包含經緯度與使用者名稱。<br>6. 系統關閉地圖對話框，顯示 SnackBar「已設定新的集合點！」。 |
| **例外情節** | **2a. 使用者無修改權限 (他人發起)：**<br>系統判定使用者無權限，鎖定地圖點擊並顯示紅字警告。使用者點選「向發起人請求修改權限」，顯示等待視窗，2 秒後給予權限。<br>**5a. 取消集合點：**<br>使用者點擊「取消集合」，系統清空本地狀態並透過 Socket 同步移除集合點 Marker。 |
| **後置條件** | 地圖上繪製紅色的集合點 Marker，群組其他成員同步收到該位置廣播。 |
