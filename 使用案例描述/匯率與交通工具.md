## 匯率與交通工具

| 欄位 | 描述內容 |
| :--- | :--- |
| **使用案例名稱** | **輸入金額進行雙向匯率換算** |
| **行動者** | 使用者、Exchange Rate API |
| **前置條件** | 系統已成功取得並暫存最新匯率表。 |
| **正常情節** | 1. 使用者點擊首頁匯率卡片，彈出「💱 匯率換算」對話框。<br>2. 使用者選擇「基準幣別」與「目標幣別」。<br>3. 使用者輸入外幣金額。<br>4. 系統觸發監聽事件，利用暫存的匯率表進行比例換算。<br>5. 系統即時在下方橘色區塊顯示換算後的總額。<br>6. 使用者點擊「上下交換圖示」，系統反轉換算方向並重新計算。 |
| **例外情節** | **前置條件 a. 初始化取得匯率失敗：**<br>若 API 超時或無網路，系統攔截錯誤並退回使用預設靜態匯率表（如 TWD:1.0, JPY:4.75），保證離線狀態的基礎運作。<br>**3a. 輸入非數字字元：**<br>系統解析字串失敗會退回 `0.0` 進行安全計算，避免程式閃退。 |
| **後置條件** | 畫面即時提供精確的小數點後第二位匯率換算結果。 |
