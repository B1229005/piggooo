## 推薦版

| 欄位 | 描述內容 |
| :--- | :--- |
| **使用案例名稱** | **一鍵匯入他人行程計畫** |
| **行動者** | 使用者、後端伺服器 |
| **前置條件** | 使用者已登入系統，正在瀏覽推薦版中的行程貼文，且已建立至少一項群組行程計畫。 |
| **正常情節** | 1. 使用者在推薦版被某篇私房行程吸引，點擊進入檢視詳細的行程與景點。<br>2. 使用者點擊畫面下方的「一鍵匯入」按鈕。<br>3. 系統跳出彈出視窗（Dialog），透過Provider獲取使用者當前的「旅遊計畫列表」供其選擇。<br>4. 使用者選定目標計畫（如：家族旅遊）後確認送出。<br>5. 系統自動將該貼文的非結構化行程圖文轉換為資料庫可辨識的標準JSON格式。<br>6. 系統將新景點寫入對應計畫的資料庫節點，並跳出「匯入成功」提示。 |
| **例外情節** | **4a. 使用者尚無任何計畫：**<br>系統偵測到計畫列表為空，自動跳出提示詢問：「您目前尚未建立任何計畫，是否立即為您創立一個新計畫並匯入？」引導使用者一鍵建立新行程。 |
| **後置條件** | 喜愛的行程順利同步寫入使用者的個人計畫中，使用者可至【計畫】分頁直接進行後續編輯。 |
