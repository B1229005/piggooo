## 交流版

| 欄位 | 描述內容 |
| :--- | :--- |
| **使用案例名稱** | **發布即時提問與AI輔助回覆** |
| **行動者** | 使用者、AI模組、後端伺服器 |
| **前置條件** | 使用者已登入系統，並處於「交流版」的即時互助論壇。 |
| **正常情節** | 1. 使用者在交流版點擊「發布問題」，輸入在地疑難雜症（如：特定景點今日是否因雨不開放）並送出。<br>2. 系統將問題寫入Firestore，其他附近的使用者瀏覽到該討論串並輸入回覆解答。<br>3. 當長篇回覆或多筆留言送出時，系統後台自動呼叫外部AI API。<br>4. AI 模組對大量非結構化的網友留言進行快速統整與事實查證，比對即時天氣與店家營業資訊。<br>5. AI產生一段精簡的摘要與「事實查證標籤」（如：營業時間已確認、天氣警報提示）。<br>6. 系統將AI摘要與驗證結果標記在討論串頂端，供提問者快速閱讀。 |
| **例外情節** | AI API 呼叫額度超限或無回應：<br>當後端呼叫OpenAI發生異常時，系統啟動安全例外處理機制，略過AI摘要步驟，直接呈現正常的網友文字留言，確保核心論壇功能不閃退、不卡死。 |
| **後置條件** | 成功產出包含網友熱心回覆與AI智能查證的即時討論串，降低使用者的資訊篩選負擔。 |
