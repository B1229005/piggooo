## 群組記帳

| 欄位 | 描述內容 |
| :--- | :--- |
| **使用案例名稱** | **設定分攤方式 (平均/精確) 並新增帳務** |
| **行動者** | 使用者、後端伺服器 |
| **前置條件** | 使用者已點擊「記一筆」開啟記帳表單。 |
| **正常情節** | 1. 使用者填寫品項名稱、總金額，並於下拉選單選擇付款人。<br>2. 使用者點擊進入「進階分攤設定」頁面。<br>3. 使用者選擇「平均分攤」，系統依據勾選人數自動將總金額除以人數。<br>4. 使用者點擊「確認」回到主表單。<br>5. 使用者點擊「新增」，系統將時間轉為 ISO 格式並連同金額分配資料打包為 JSON。<br>6. 系統發送 POST 請求至後端 API。<br>7. 系統重新載入歷史紀錄與結餘，並關閉表單。 |
| **例外情節** | **3a. 精確分攤金額不符：**<br>若切換至「精確分攤」並手動輸入各成員金額，若加總不等於總額，畫面顯示「剩餘 NT$ X」或「超額」，並阻擋提交。<br>**6a. API 傳輸失敗：**<br>系統阻擋關閉表單，並透過 SnackBar 顯示：「⚠️ 儲存失敗，請檢查後端連線！」 |
| **後置條件** | 群組帳本增加一筆新消費，UI 更新各成員的應收/付款與剩餘預算。 |
