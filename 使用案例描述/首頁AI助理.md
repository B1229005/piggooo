## 首頁AI助理

| 欄位 | 描述內容 |
| :--- | :--- |
| **使用案例名稱** | **解析指令觸發自動記帳** |
| **行動者** | 使用者、OpenAI API、後端伺服器 |
| **前置條件** | 使用者位於首頁，且已設定有效的 OpenAI API Key。 |
| **正常情節** | 1. 使用者點擊首頁右下角的懸浮按鈕，開啟 AI 聊天對話框。<br>2. 使用者輸入語意指令，例如：「幫我記一杯手搖飲 50 元，大家平分」。<br>3. 系統將當前情境（包含：天氣、GPS位置、群組成員名單、結餘）與對話紀錄發送至 OpenAI API。<br>4. OpenAI API 判斷需呼叫工具，回傳 `add_expense` Function Call。<br>5. 系統解析 JSON 參數，抓取出總金額、付款人與分攤者名單。<br>6. 系統轉換成員 ID，自動計算平分金額。<br>7. 系統發送 POST 請求至後端儲存帳務。<br>8. 儲存成功，聊天室顯示：「✅ 記好囉！算在大家的帳上！💸」並更新首頁 UI。 |
| **例外情節** | **4a. API 將指令視為一般對話：**<br>若使用者指令不明確，API 未觸發工具呼叫，系統將直接以 Markdown 格式渲染 AI 的一般文字回覆。<br>**7a. 後端伺服器記帳失敗：**<br>系統捕捉例外，並在聊天室顯示：「記帳失敗了，請檢查一下網路連線！🐷」 |
| **後置條件** | 成功新增一筆帳單，群組結餘自動重新計算。 |
